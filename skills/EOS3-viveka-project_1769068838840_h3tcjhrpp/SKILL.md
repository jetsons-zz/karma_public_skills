# 文章助手

专业的多平台内容创作与发布工具。根据主题或参考文章智能生成高质量文章内容,自动配图,并针对微信公众号、知乎、小红书、X、Discord、微博、知识星球等平台的爆款规则进行内容优化。生成后自动插入图片进行精美排版,最终以HTML格式输出。使用触发词「文章助手」或「多平台内容创作」「洗文章」启动完整工作流。当用户需要创作适配多平台的文章、根据主题生成配图内容、或将内容发布到社交媒体平台时激活

## 使用场景

当用户需要创作适配多个社交平台的文章内容、根据主题生成配图、或将内容发布到微信公众号、知乎、小红书、X、Discord等平台时激活

**触发关键词:** 创作文章, 撰写文章, 多平台内容, 文章配图, HTML文章

**激活模式:**
- `根据.*生成文章`
- `写.*文章`
- `创作.*内容`
- `生成.*HTML`

## 使用说明

=== 文章助手技术参考 ===

[核心功能]

1. 内容生成引擎
- 基于主题的原创文章生成
- 参考文章改写与优化
- 多种写作风格适配
- 自动段落结构优化

2. 平台适配规则

微信公众号:
- 标题: 15-20字,包含数字/疑问/对比
- 正文: 1500-3000字,3-5个小标题
- 配图: 每300-500字配1张图,首图16:9
- 排版: 两端对齐,行间距1.75,段间距15px

知乎:
- 标题: 问题式表达,20-30字
- 正文: 2000-4000字,逻辑清晰
- 配图: 支持GIF,图片清晰度高
- 特色: 数据支撑,专业术语,引用来源

小红书:
- 标题: 关键词,吸睛短句
- 正文: 500-1000字,口语化表达
- 配图: 首图吸睛,9宫格排版
- 标签: #话题标签 3-5个

X(Twitter):
- 标题: 简洁有力,280字符限制
- 正文: 分thread展开
- 配图: 支持4张图,16:9或1:1
- 特色: hashtag策略,@提及

Discord:
- 标题: Markdown格式支持
- 正文: 代码块友好
- 配图: 支持嵌入链接
- 特色: embed格式,多媒体支持

微博:
- 标题: 热点结合,140字以内
- 正文: 简短精炼
- 配图: 最多9张,正方形
- 特色: 话题标签,@好友

知识星球:
- 标题: 价值导向
- 正文: 1000-2000字,干货为主
- 配图: 高质量配图
- 特色: 深度内容,专业性强

3. 图片生成与配图

- 使用AI图片生成API(推荐Gemini Image API)
- 根据段落主题自动生成配图提示词
- 图片尺寸自适应平台要求
- 支持自定义图片风格和色调

4. HTML排版系统

基础结构:
<!DOCTYPE html>
<html>
<head>
  <meta charset='UTF-8'>
  <meta name='viewport' content='width=device-width, initial-scale=1.0'>
  <title>文章标题</title>
  <style>
    /* 响应式样式 */
  </style>
</head>
<body>
  <article>
    <header>
      <h1>标题</h1>
      <div class='meta'>作者 | 日期</div>
    </header>
    <section>
      <!-- 文章内容 -->
    </section>
  </article>
</body>
</html>

CSS样式要点:
- 响应式布局: max-width 800px, margin auto
- 字体: 正文16px, 行高1.8
- 颜色: 主色调配合平台风格
- 图片: width 100%, height auto, border-radius
- 段间距: margin 20px 0

5. 工作流程

Step 1: 内容分析
- 解析用户输入(主题/参考文章)
- 确定目标平台
- 分析平台爆款规则

Step 2: 文章生成
- 构建文章大纲
- 生成标题(多平台变体)
- 撰写正文内容
- 优化段落结构

Step 3: 配图生成
- 提取关键段落
- 生成图片提示词
- 调用图片生成API
- 保存图片到本地

Step 4: HTML排版
- 插入图片到对应位置
- 应用CSS样式
- 添加平台特定元素(标签/链接等)
- 生成最终HTML文件

Step 5: 输出与预览
- 保存HTML文件
- 提供预览链接
- 输出各平台适配版本

[API集成]

图片生成API(Gemini):
- Endpoint: generativelanguage.googleapis.com
- 模型: gemini-2.0-flash-image-exp
- 参数: prompt, aspectRatio, responseFormat

[最佳实践]

1. 标题优化:
- 包含关键词前置
- 使用数字增强可信度
- 制造悬念或提出问题
- 长度控制在平台最佳范围

2. 内容结构:
- 开头100字吸引注意
- 小标题层次清晰
- 每段3-5行为宜
- 结尾呼应开头或行动号召

3. 配图策略:
- 首图最重要,决定点击率
- 配图与段落主题强相关
- 保持视觉风格统一
- 避免文字过多的图片

4. SEO优化:
- 关键词密度2-3%
- 使用语义化HTML标签
- 添加meta描述
- 图片alt属性填写

[性能优化]

- 图片压缩: 保持质量前提下压缩至100-300KB
- 懒加载: 图片延迟加载提升首屏速度
- CDN加速: 图片资源使用CDN
- 缓存策略: 静态资源设置缓存

[注意事项]

- 尊重原创,标注引用来源
- 避免敏感话题和违规内容
- 检查平台内容规范
- 测试多设备兼容性
- 保存备份版本

## 常见问题

**问: 如何配置图片生成API密钥?**

答: 需要在Gemini AI Studio获取API密钥。步骤: 1)访问 ai.google.dev 2)创建项目 3)启用Gemini API 4)生成API密钥 5)将密钥配置到技能设置中。首次使用时会提示配置向导。

**问: 生成的图片质量不理想怎么办?**

答: 可以通过以下方式优化: 1)提供更详细的配图风格描述 2)在prompt中添加'high quality, professional'等关键词 3)调整aspectRatio参数 4)尝试不同的图片生成模型 5)使用更具体的场景描述而非抽象概念。

**问: 如何自定义HTML样式?**

答: 有三种方式: 1)使用内置模板(wechat/zhihu/xiaohongshu等) 2)修改templates目录下的HTML文件 3)在生成时指定自定义CSS。可以通过'使用[模板名]风格'来指定,或提供自己的CSS代码。

**问: 支持哪些平台的爆款规则?**

答: 目前支持: 微信公众号、知乎、小红书、X(Twitter)、Discord、微博、知识星球。每个平台都有针对性的标题优化、内容结构、配图策略和排版规范。可以一次生成多平台适配版本。

**问: 生成文章需要多长时间?**

答: 通常2-5分钟完成,具体取决于: 1)文章字数(500-4000字) 2)配图数量(1-9张) 3)目标平台数量 4)网络状况。图片生成是最耗时的环节,每张图约需10-20秒。

**问: 如何确保内容原创性?**

答: 系统通过以下方式保证原创: 1)基于主题重新组织观点而非简单复制 2)使用不同表达方式和案例 3)改写时保留核心思想但更换语言风格 4)建议使用查重工具二次检查 5)标注引用来源。

**问: 生成的HTML如何发布到平台?**

答: 发布方式: 1)微信公众号: 复制HTML到编辑器或使用秀米等工具 2)知乎: 直接粘贴或使用Markdown转换 3)小红书: 需要手动复制文字和上传图片 4)其他平台: 根据平台支持的格式调整。建议先在本地预览HTML效果。

**问: 可以批量生成多篇文章吗?**

答: 支持批量生成。可以: 1)提供主题列表一次性生成多篇 2)指定相同主题的不同角度 3)为同一内容生成多平台版本。建议每次不超过5篇以保证质量。批量生成会按顺序处理,总时间为单篇时间x数量。

## 调试指南

[调试指南]

常见问题排查:

1. API密钥错误
错误信息: 'API key invalid'
解决方案: 检查Gemini API密钥是否正确配置,确保密钥有效且未过期

2. 图片生成失败
错误信息: 'Image generation timeout'
解决方案: 检查网络连接,尝试简化prompt,或降低图片数量

3. HTML样式异常
错误信息: 样式显示不正确
解决方案: 检查模板文件是否完整,确认CSS语法正确,测试不同浏览器兼容性

4. 平台适配问题
错误信息: 内容不符合平台规范
解决方案: 查看平台最新内容规范,调整字数和格式要求

5. 内容质量不佳
错误信息: 生成内容不满意
解决方案: 提供更详细的需求描述,使用参考文章,明确风格要求

日志查看:
- 生成日志保存在 logs/article-generation.log
- API调用日志保存在 logs/api-calls.log
- 错误日志保存在 logs/errors.log

性能优化:
- 减少不必要的图片生成
- 使用缓存避免重复API调用
- 控制单次生成的文章数量
- 优化图片大小和格式

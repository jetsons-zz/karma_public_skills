# 政务文书助理

专业的政务文书撰写助手，严格遵循《党政机关公文处理工作条例》和《党政机关公文格式》国家标准（GB/T 9704-2012），支持心得体会、通知、讲话稿、工作报告、请示、工作方案、调研报告、邀请函八大类文书的撰写。提供标准公文格式输出、规范政务语言体系和多元输出能力（Word/PDF/Markdown）。当用户说'写通知'、'起草讲话稿'、'写工作报告'、'撰写请示'、'制定工作方案'、'写心得体会'、'调研报告'、'邀请函'时触发。

## 使用场景

当用户需要撰写符合党政机关公文规范的各类政务文书，包括通知、讲话稿、工作报告、请示、工作方案、心得体会、调研报告、邀请函等文件时，或需要将内容发布到政府部门使用时激活

**触发关键词:** 写通知, 起草讲话稿, 写工作报告, 撰写请示, 制定工作方案, 写心得体会, 调研报告, 邀请函, 政务文书, 公文写作, 党政机关公文, 政府文件

**激活模式:**
- `(写|撰写|起草|制定).*(通知|讲话稿|工作报告|请示|工作方案|心得体会|调研报告|邀请函)`
- `(政务|公文|政府).*(文书|写作|文件)`

# 政务文书助理参考文档

## 功能说明

政务文书助理是面向政府部门的专业文书撰写工具，支持八大类公文撰写：

1. 心得体会：学习培训后的思想总结材料
2. 通知：政策发布、事项传达类文件
3. 讲话稿：领导会议发言专用文稿
4. 工作报告：向上级汇报工作进展与成果
5. 请示：向上级请求批准事项的正式文件
6. 工作方案：具体工作的实施计划与安排
7. 调研报告：专项调研后的分析总结材料
8. 邀请函：政务场合的正式邀约文书

## 技术规格

### 权威依据

- 《党政机关公文处理工作条例》（2012年）
- 《党政机关公文格式》国家标准（GB/T 9704-2012）
- 通用政务写作规范与惯例

### 公文格式标准

#### 版式要求
- 纸张：A4纸（210mm×297mm）
- 页边距：上37mm，下35mm，左28mm，右26mm
- 行距：固定值28磅
- 字间距：标准

#### 字体规范
- 标题：方正小标宋简体，二号字
- 正文：仿宋GB2312，三号字
- 发文字号：仿宋GB2312，三号字
- 附件说明：仿宋GB2312，三号字
- 成文日期：仿宋GB2312，三号字
- 主送机关：仿宋GB2312，三号字
- 版记：仿宋GB2312，四号字

#### 公文要素
1. 版头：份号、密级、紧急程度、发文机关标志、发文字号、签发人
2. 主体：标题、主送机关、正文、附件说明、发文机关署名、成文日期、印章
3. 版记：抄送机关、印发机关、印发日期、页码

### 政务语言规范

#### 常用政务术语
- 贯彻落实、深入推进、统筹协调、扎实开展
- 高度重视、充分认识、切实加强、积极探索
- 全面部署、精心组织、狠抓落实、确保实效

#### 表述方式
- 开头：根据、按照、为了、经研究决定
- 过渡：同时、此外、另外、与此同时
- 结尾：特此通知、请认真贯彻执行、妥否请批示

## 最佳实践

### 撰写流程

1. 明确文书类型和主题
2. 收集相关政策文件和背景资料
3. 确定发文单位和收文对象
4. 使用助手生成初稿
5. 人工审核和完善内容
6. 格式校验和排版
7. 导出正式文档

### 质量控制

- 政治性：确保政治立场正确，符合党的路线方针政策
- 准确性：事实准确，数据可靠，引用规范
- 规范性：格式标准，用语得体，逻辑清晰
- 时效性：及时反映最新政策精神和工作要求

## 注意事项

1. 严格遵守保密纪律，不泄露涉密信息
2. 标题要准确概括主题，简明扼要
3. 主送机关要准确无误，层级对应
4. 正文结构要清晰，层次分明
5. 成文日期要规范，用汉字书写
6. 附件要与正文内容相符
7. 印章要端正盖在成文日期上方
8. 页码要连续编排，居中标注

## 示例

### 撰写部门工作通知

**场景:** 政府部门需要向下级单位或相关企业发布工作任务

**用户输入:**
```
帮我写一份关于开展安全生产检查的通知，发文单位是市安监局，主送各区县安监局和重点企业，要求在本月底前完成自查。
```

**预期输出:**
```
生成标题为'关于开展安全生产检查工作的通知'的正式公文，包含主送机关、通知事项（检查范围、时间要求、重点内容）、工作要求（自查整改、上报材料、联系方式）、结束语'特此通知'，落款为市安监局和成文日期，格式符合GB/T 9704-2012标准。
```

### 起草领导讲话稿

**场景:** 领导需要在重要会议上发表讲话

**用户输入:**
```
写一份在全市经济工作会议上的讲话稿，主题是总结去年经济工作成绩，部署今年重点任务，要求3000字左右。
```

**预期输出:**
```
生成标题为'在全市经济工作会议上的讲话'的讲话稿，包含称谓（同志们）、开场白、主体内容（一是回顾去年工作亮点和数据，二是分析当前经济形势，三是明确今年工作目标，四是部署重点任务，五是提出工作要求）、结束语，语言庄重有力，逻辑严密，符合领导讲话风格。
```

### 编写工作方案

**场景:** 需要制定专项工作的实施计划

**用户输入:**
```
制定一个关于推进政务服务数字化转型的工作方案，包括指导思想、目标任务、实施步骤和保障措施。
```

**预期输出:**
```
生成标题为'推进政务服务数字化转型工作方案'的完整方案，包含：一、指导思想（引用相关政策精神）；二、工作目标（总体目标和具体指标）；三、主要任务（分5-6项重点工作）；四、实施步骤（分阶段安排时间节点）；五、保障措施（组织领导、资金保障、技术支撑、督查考核）；六、工作要求（提高认识、加强协作、确保实效），落款为制定单位和日期。
```

### 撰写请示文件

**场景:** 下级单位需要向上级申请资源或批准事项

**用户输入:**
```
写一份向市政府请示增加专项资金的文件，我们是市教育局，需要申请500万元用于改善农村学校办学条件。
```

**预期输出:**
```
生成标题为'关于申请增加农村学校办学条件改善专项资金的请示'的上行文，主送机关为市人民政府，正文包含：一、请示缘由（农村学校现状和问题）；二、资金用途（具体项目和预算明细）；三、预期效果（改善情况和受益学生数）；四、结束语'妥否请批示'，落款为市教育局和日期，格式规范，理由充分，数据详实。
```

### 撰写学习心得体会

**场景:** 参加学习培训后需要撰写个人体会

**用户输入:**
```
写一篇参加党的二十大精神学习培训班的心得体会，800字左右，要结合自己的工作实际。
```

**预期输出:**
```
生成标题为'学习党的二十大精神心得体会'的个人总结材料，包含：引言（培训背景和基本情况）；主体部分分三点论述（一是对二十大报告核心内容的认识和理解，二是联系本职工作的思考和启发，三是今后工作中如何贯彻落实）；结尾（表决心和努力方向），落款为单位、姓名、日期，语言真实具体，有感而发，政治站位正确。
```

## 常见问题

**问: 生成的公文格式不符合国标要求怎么办？**

答: 请检查以下几点：1）确认使用了正确的文书类型；2）运行格式检查器脚本进行自动检测；3）对照GB/T 9704-2012标准手动调整页边距、字体字号、行距等参数；4）查看参考文档中的格式规范说明；5）使用提供的标准模板重新生成。

**问: 如何确保公文的政治正确性？**

答: 建议采取以下措施：1）使用政务术语库中的标准表述；2）引用权威政策文件和领导讲话精神；3）避免使用口语化、网络化表达；4）涉及重要政治概念时使用全称（如'习近平总书记'）；5）生成后务必由专业人员进行政治审核；6）参考近期同类公文的表述方式。

**问: 可以导出哪些格式的文档？**

答: 目前支持三种格式导出：1）Word格式（.docx）：完全符合公文格式标准，可直接打印盖章；2）PDF格式（.pdf）：适合电子传阅和归档；3）Markdown格式（.md）：便于在线编辑和版本控制。导出时会自动应用标准字体、字号、行距等格式参数。

**问: 生成的内容过于模板化，缺乏针对性怎么办？**

答: 提高内容质量的方法：1）提供更详细的背景信息和具体数据；2）明确指出重点内容和特殊要求；3）提供参考资料或同类文书样本；4）生成后进行人工润色和补充；5）使用'请详细展开...'等指令要求深化某部分内容；6）结合本单位实际情况进行个性化调整。

**问: 如何处理涉密信息？**

答: 处理涉密信息的注意事项：1）不要在输入中包含任何涉密内容；2）仅提供框架性、原则性信息；3）生成后在本地填充具体涉密数据；4）按照保密规定标注密级和份号；5）通过内网或加密渠道传输；6）严格执行保密管理制度。切记：本工具不具备涉密信息处理资质。

**问: 不同地区或部门的公文格式有差异怎么办？**

答: 应对格式差异的方法：1）本工具严格遵循国家标准GB/T 9704-2012；2）各地方或部门的特殊要求可在生成后手动调整；3）保存本单位的格式偏好设置；4）制作符合本单位要求的自定义模板；5）咨询本单位办公室或文秘部门获取标准范本；6）建立本单位公文格式规范文档。

# 调试指南

## 常见问题排查步骤

### 1. 功能不响应
- 检查触发词是否准确（如'写通知'、'起草讲话稿'）
- 确认是否提供了必要的信息（文书类型、主题等）
- 查看工具权限是否已授权（Read、Write等）
- 重新描述需求，使用更明确的表述

### 2. 生成内容不符合预期
- 提供更详细的背景信息和具体要求
- 明确指出重点内容和字数要求
- 参考使用示例，调整输入方式
- 分步骤生成：先生成框架，再补充细节

### 3. 格式问题
- 运行格式检查器脚本：python scripts/format_checker.py
- 对照参考文档中的格式规范
- 使用标准模板重新生成
- 检查字体、字号、行距等参数设置

### 4. 导出失败
- 确认有文件写入权限
- 检查目标路径是否存在
- 验证文档内容是否包含特殊字符
- 尝试更换导出格式

### 5. 脚本执行错误
- 确认Python版本>=3.9
- 检查依赖包是否已安装
- 查看错误日志定位问题
- 验证输入参数格式是否正确

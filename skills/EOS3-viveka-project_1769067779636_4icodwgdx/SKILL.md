# 产品历史海报

基于用户提供的产品名称，自动生成产品发展历史的博物馆级3D信息图海报（16:9）。使用Gemini Image生成API创建包含完整时间线、技术规格、材质演变的高密度视觉设计。支持任意产品类型，从消费电子到潮流玩具。触发词：调用产品历史海报、生成产品历史海报、创建产品发展史

当用户需要生成展示产品历史发展的视觉海报、信息图或时间线图表时激活

**激活模式:**
- `生成.*产品.*历史.*海报`
- `创建.*发展史.*海报`
- `.*产品.*时间线`

## 使用说明

=== 产品历史海报技能参考文档 ===

[核心技术]
本技能基于Google Gemini Image Generation API实现，使用Gemini 2.0 Flash Image模型生成高质量产品历史海报。

[提示词结构]
生成的提示词包含以下核心模块：
1. 角色与主题定义：16:9 3D信息图海报，博物馆级产品摄影风格
2. 英雄系列（时间线）：8-12个历史版本的产品，按时间顺序排列
3. 品牌氛围：深邃的品牌色背景，水印专利图纸和历史元素
4. 超密集信息层（PUNCH风格）：包含注释网络、情境区、放大镜头、技术规格条
5. 技术参数：Octane渲染、虚幻引擎5美学、体积光照

[API调用流程]
1. 接收用户输入的产品名称
2. 询问用户品牌色彩偏好和材质风格（可选）
3. 构建结构化提示词，替换[产品名称]、[品牌色彩]、[Material Vibe]等变量
4. 调用Gemini Image API生成16:9比例的高分辨率海报
5. 返回生成的图像并提供下载选项

[提示词优化技巧]
- 使用'museum-grade product photography'确保质感
- 强调'timeline'、'chronological order'保证时间顺序
- 添加'hyper-realistic 3D'、'8K resolution'提升细节
- 使用'PUNCH style'、'data aesthetics'创建信息密集效果
- 指定'volumetric lighting'、'sharp focus'增强视觉冲击力

[变量说明]
- [产品名称]：用户输入的核心产品（如'泡泡玛特'、'iPhone'、'特斯拉'）
- [品牌色彩]：品牌主色调（如'teal'、'space gray'、'red'）
- [Material Vibe]：早期材质风格（如'plastic'、'metal'、'wood'）
- [关键组件]：产品特征部件（如'curves'、'buttons'、'engine'）

## 常见问题

**问: 生成的海报中产品版本数量不够怎么办？**

答: 可以在提示词中明确指定数量，如'precisely 12 versions spanning from 2008 to 2024'。如果产品历史较短，可以包含概念设计或未来预测版本来填充时间线。

**问: 如何确保生成的海报符合品牌视觉规范？**

答: 在调用时明确指定品牌色（使用具体色彩名称或hex值），并在提示词中添加品牌关键词，如'Apple minimalist aesthetic'或'Tesla futuristic design language'。可以在negativePrompt中排除不符合品牌调性的元素。

**问: 生成的图像信息密度不够或过于拥挤？**

答: 调整提示词中的'ULTRA-DENSE'描述强度。信息过少时，增加'hundreds of annotations'、'complex data layers'等描述；过于拥挤时，使用'organized information hierarchy'、'clean margins'来平衡。

**问: 如何处理没有明确历史版本的新产品？**

答: 可以调整为'产品线海报'模式，展示同一品牌下的不同产品型号，或者创建'概念演变'海报，展示从草图到成品的设计过程。

**问: 生成的海报语言不是中文怎么办？**

答: Gemini Image主要基于视觉生成，文字部分可能显示英文。如需中文标题和注释，可以在后期使用图像编辑工具添加，或在提示词中明确要求'Chinese text labels and annotations'。

**问: API调用失败或返回错误？**

答: 常见原因：1) API密钥未配置或过期；2) 提示词包含违禁内容触发安全过滤；3) 超出配额限制。检查环境变量GEMINI_API_KEY，查看API响应中的error字段获取详细错误信息。

## 调试指南

[调试指南]

1. API连接问题
   - 检查环境变量：echo $GEMINI_API_KEY
   - 验证API访问：curl -H 'x-goog-api-key: YOUR_KEY' https://generativelanguage.googleapis.com/v1beta/models/gemini-2.0-flash-exp
   - 确认网络连接和代理设置

2. 提示词优化
   - 使用--verbose标志查看完整提示词
   - 如果结果不理想，尝试调整关键词强度（增加描述性形容词）
   - 使用negativePrompt排除不想要的元素
   - 参考examples中的成功案例

3. 图像质量问题
   - 确保使用gemini-2.0-flash-exp或更高版本模型
   - 检查aspectRatio是否设置为16:9
   - 如果分辨率不足，在提示词中强调'8K resolution'、'ultra high definition'

4. 内容安全过滤
   - 如果触发安全过滤，检查产品名称是否包含敏感词
   - 调整safetySettings参数（谨慎使用）
   - 使用更中性的描述替换可能引发过滤的词汇

5. 性能优化
   - 单次生成建议设置numberOfImages为1
   - 使用缓存机制避免重复生成相同产品
   - 异步处理多个生成请求
